01 / 서비스 구조
데이터 수집부터 AI 답변 생성까지 전체 흐름
옵타 데이터 수신 → DB 적재 전처리 → LIVE 스코어 DB 저장 → AI 입력용 전처리 → 프롬프트 조립 → LLM API 호출 → 답변 생성
02 / 테스트 플로우
버기 버튼 3개 × 멀티 모델 병렬 비교 및 기획자 채점
채팅형이 아닌 계산기형 UX. 배터들이 공통으로 필요로 하는 질문 패턴 3개를 버튼으로 정의. 버튼 하나 = 사전 정의된 질문 1개. 각 버튼은 독립적으로 모든 LLM 모델에 동시 전송되어 답변을 비교.
03 / 개선 방안
옵타 직접 연결 + LLM 2트랙 전략
옵타 원본 피드를 LLM에 직접 연결. 1순위는 외부 LLM으로 퀄리티 우선 승부. 2순위는 오픈소스 기반 스포츠 배팅 특화 내부 LM 자체 개발. 처음부터 외부 → 내부 대체 가능한 구조로 설계.
